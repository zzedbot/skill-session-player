# 🔐 Session Player 安全认证指南

**版本**: 1.2.0  
**更新日期**: 2026-03-06

---

## 📋 概述

为了保护您的会话录制数据安全，Session Player 现已启用邮箱验证码登录认证系统。只有通过验证的用户才能访问录制库和播放器。

---

## 🔒 安全特性

### 1. 邮箱白名单
- ✅ **仅允许** `geolle@163.com` 登录
- ❌ 其他邮箱地址将被拒绝

### 2. 一次性验证码
- 6 位数字验证码
- 5 分钟有效期
- 使用后立即失效
- 防止重放攻击

### 3. 会话管理
- 会话有效期：7 天
- 自动清理过期会话
- 支持手动登出
- Cookie 安全设置（SameSite=Strict）

### 4. 邮件安全
- 使用 SSL/TLS 加密传输
- 邮件模板包含安全提示
- 验证码不存储在数据库中（内存存储）

## 🚀 使用流程

### 步骤 1: 访问登录页面

**本地访问**: http://localhost:3000/session-player/login.html  
**互联网访问**: https://<your-domain>/session-player/login.html

### 步骤 2: 输入邮箱

您好！

您正在登录 Session Player 会话回放系统，请使用以下验证码完成登录：

123456

验证码说明：
- 有效期：5 分钟
- 仅限本次登录使用
- 不要分享给任何人

### 步骤 4: 输入验证码

- 输入 6 位数字验证码
- 点击"🔐 登录"按钮
- 或使用回车键快速提交

### 步骤 5: 登录成功

- 自动跳转到录制库首页
- 右上角显示登录邮箱

## 📧 邮件发送配置

### SMTP 设置（配置文件）

编辑 `config/config.json`：

```json
{
  "email": {
    "smtp": {
      "host": "smtp.163.com",
      "port": 465,
      "secure": true,
      "user": "zedbot@163.com",
      "pass": "YOUR_SMTP_PASSWORD"
    }
  }
}
```

> ⚠️ **安全**: 不要将真实密码提交到 Git！`config/config.json` 已在 `.gitignore` 中。

### 邮件发送失败处理

**可能原因**:
1. SMTP 密码错误
2. 网络连接问题
3. 邮箱服务器限制
4. 发送频率过高

**解决方案**:
```bash
# 检查服务器日志
cd /zed/workspace/skills/session-player
npm start

# 查看控制台输出
# 寻找 "✅ 验证码邮件已发送" 或 "❌ 发送邮件失败"
```

## 🐛 故障排查

### 问题 1: 收不到验证码邮件

**检查步骤**:
1. 检查垃圾邮件箱
2. 确认邮箱地址正确（geolle@163.com）
3. 查看服务器日志
4. 检查 SMTP 配置

**解决方案**:
```bash
# 测试邮件发送
node -e "
const auth = require('./auth');
auth.sendVerificationCode('geolle@163.com')
    .then(() => console.log('✅ 发送成功'))
    .catch(err => console.error('❌ 发送失败:', err.message));
"
```

### 问题 2: 验证码错误

**可能原因**:
1. 输入错误
2. 验证码已过期（>5 分钟）
3. 验证码已使用过

**解决方案**:
- 点击"重发"按钮获取新验证码
- 等待 60 秒冷却时间后重试

### 问题 3: 登录后仍然无法访问

**可能原因**:
1. Cookie 被禁用
2. 浏览器缓存问题
3. 会话已过期

**解决方案**:
```bash
# 清除浏览器缓存和 Cookie
# 或使用无痕模式重新登录

# 检查浏览器控制台
# F12 -> Console -> 查看错误信息
```

### 问题 4: 登出后仍然可以访问

**可能原因**:
1. 浏览器缓存
2. Cookie 未清除

**解决方案**:
- 清除浏览器缓存
- 手动删除 Cookie
- 使用无痕模式测试

## 📝 技术实现

### 认证流程
```
用户请求登录
    ↓
输入邮箱（geolle@163.com）
    ↓
生成 6 位验证码
    ↓
发送验证码邮件
    ↓
用户输入验证码
    ↓
验证验证码（正确性 + 有效期）
    ↓
创建会话（7 天有效期）
    ↓
返回 sessionId（Cookie）
    ↓
后续请求携带 sessionId
    ↓
验证会话有效性
    ↓
允许/拒绝访问
```

### 数据存储

**内存存储**（重启后清除）:
- `verificationCodes`: Map<email, {code, expiresAt}>
- `activeSessions`: Map<sessionId, {email, createdAt}>

**Cookie 存储**:
- `session_id`: 会话 ID
- 有效期：7 天
- SameSite: Strict
